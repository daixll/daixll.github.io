# 高质量代码

高质量代码

* 正确可靠
  * 边界条件是否考虑完备
  * 异常情况处理，稳定性保证

* 简洁清晰
  * 易读易维护

错误处理
* 简单错误
  * 在其他地方不需要捕获这个错误
  * 优先使用 `errors.New` 直接表示简单错误
  
* 错误的 wrap（包）
  * 当嵌套调用多个函数的时候，error嵌套另一个error，需要生成一个跟踪链
  * 优先使用 `fmt.Errorf` 中的 `%w` 来将错误关联至错误链中

* 错误判定
  * 在错误链上获取特定种类的错误
  * 优先使用 `errors.As`
